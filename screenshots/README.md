## 🎯 How to Add Screenshots

### Method 1: Direct File Upload
1. Take screenshots of your application
2. Save them with the exact filenames listed above
3. Place them in this `screenshots/` folder
4. The README.md will automatically display them

### Method 2: Using GitHub Issues (Alternative)
1. Create a GitHub issue in your repository
2. Drag and drop screenshots into the issue comment
3. Copy the generated markdown links
4. Replace the local paths in README.md with the GitHub URLs

### Method 3: Using External Image Hosting
- Upload images to services like:
  - [Imgur](https://imgur.com/)
  - [Cloudinary](https://cloudinary.com/)
  - [GitHub Issues](https://github.com/yourusername/clinicconnect/issues)

## 📏 Recommended Image Specifications

- **Format**: PNG or JPG
- **Width**: 1200px - 1920px
- **Aspect Ratio**: 16:9 or 4:3
- **File Size**: Under 2MB per image
- **Quality**: High resolution for clarity

## 🎨 Screenshot Guidelines

### What to Capture:
1. **Landing Page**: Show the main hero section with search functionality
2. **Doctor Search**: Display the search results and filtering options
3. **Appointment Booking**: Show the booking form and time slot selection
4. **AI Chatbot**: Capture the chat interface in action
5. **User Profile**: Show profile management and settings
6. **Doctor Dashboard**: Display the main dashboard with appointments
7. **Admin Panel**: Show the admin interface and analytics
8. **Mobile View**: Demonstrate responsive design on mobile

### Tips:
- Use a clean, uncluttered browser window
- Hide personal information in screenshots
- Show the most important features prominently
- Use consistent styling and branding
- Include both desktop and mobile views

## 🔄 Updating Screenshots

When you update the application:
1. Take new screenshots
2. Replace the old files with the same names
3. Update the README.md descriptions if needed
4. Commit and push the changes
